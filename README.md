# Disk Scheduling Simulator

## Overview

Disk scheduling is a critical operating system function that determines the order in which disk I/O requests are serviced. The efficiency of a disk scheduling algorithm directly impacts system performance by minimizing seek time and maximizing disk throughput.

This project implements and simulates several classic disk scheduling algorithms, allowing users to observe their behavior, compare their performance, and analyze the total head movement generated by each strategy.

### Implemented Algorithms

#### 1. FCFS (First-Come, First-Served)

Requests are serviced in the order they arrive. This algorithm is simple and fair but may result in poor overall performance due to excessive head movement.

#### 2. SSTF (Shortest Seek Time First)

The next request serviced is the one closest to the current disk head position. This generally reduces seek time but may lead to starvation of distant requests.

#### 3. SCAN

The disk head moves in one direction, servicing all requests along the way. Upon reaching the end of the disk, it reverses direction and continues servicing requests.

#### 4. C-SCAN (Circular SCAN)

The disk head moves in only one direction while servicing requests. After reaching cylinder 0 (the lowest cylinder boundary), it jumps directly back to the highest pending request and continues scanning.

#### 5. LOOK

Similar to SCAN, but instead of traveling all the way to the disk boundary, the head only moves as far as the last pending request before reversing direction.

#### 6. C-LOOK (Circular LOOK)

Similar to C-SCAN, but the head only travels as far as the last pending request in the current direction. After servicing it, the head jumps directly to the lowest pending request and continues.

## Usage

The project exposes a command-line entry point:

```bash
run-simulator
```

### Required Arguments

| Argument | Long Form     | Description                                         |
| -------- | ------------- | --------------------------------------------------- |
| `-p`     | `--position`  | Initial position of the disk head                   |
| `-d`     | `--direction` | Initial direction of the disk head (`inc` or `dec`) |
| `-s`     | `--size`      | Total number of cylinders in the disk               |
| `-q`     | `--queue`     | Queue of disk requests                              |

### Example

```bash
run-simulator \
    -p 50 \
    -d inc \
    -s 199 \
    -q 98 183 37 122 14 124 65 67
```

### Argument Details

#### Initial Head Position

```bash
-p 50
```

Starts the disk head at cylinder `50`.

#### Initial Direction

```bash
-d inc
```

Possible values:

* `inc`: Move toward larger cylinder numbers.
* `dec`: Move toward smaller cylinder numbers.

#### Disk Size

```bash
-s 199
```

Specifies the maximum cylinder number available on the disk.

#### Request Queue

```bash
-q 98 183 37 122 14 124 65 67
```

A space-separated list of cylinder requests to be serviced.
